Wait...why index.html?

  • index.html is the default filename a web browser looks for when pointed to any directory.
  • You won't necessarily see it reflected in the URL — we just see http://girldevelopit.com, but the browser is displaying http://girldevelopit.com/index.html. Sneaky!
  • You should have an index.html file for every directory on your site. If not, users will see a list of all your filenames, like this:

Structural Elements!

HTML5 Element: <header>

  • Container for a group of introductory or navigational aids
  • Document can have multiple header elements
  • E.g. One for the page, one for each section

              <header>
                <a href="/"><img src="site-logo.png" /></a>

                <nav>
                  <ul>
                    <li>Your menu</li>
                  </ul>
                </nav>
              </header>

HTML5 Element: <footer>

  • Contains information about its containing element
  • E.g. who wrote it, copyright, links to related content, etc.
  • Document can have multiple footer elements
  • E.g. One for the page, one for each section

              <footer>
                <p>©2014 Girl Develop It. Super fancy codes written by Julie, Nicole and Leeann.</p>
              </footer>

HTML5 Element: <main>

  • Use once per page to mark where main content begins
  • Helpful for screen readers/assistive technology, content syndication, and mobile browsers

              <main>
                 <h1>Web Browsers</h1>
                 <p>Google Chrome, Firefox, and Internet Explorer are the most used browsers today.</p>

                 <article>
                    <h1>Google Chrome</h1>
                    <p>Google Chrome is a free, open-source web browser developed by Google.</p>
                 </article>

                 <article>
                    <h1>Mozilla Firefox</h1>
                    <p>Firefox is a free, open-source web browser from Mozilla.</p>
                 </article>
              </main>

HTML5 Element: <aside>

  • Tangentially related content
  • E.g. sidebar, pull quotes

Inline vs Block

So far, we have mostly seen "block" elements.

  • They render on new lines, ie. paragraphs.
  • Block elements: p, h1, ul, li, almost everything else

There are also "inline" elements

  • They render on the same line that they are written on.
  • Inline elements: img, a, br, em, strong

Box Model

The CSS box model is essentially a box that wraps around HTML elements, and consists of: margins, borders, padding, and the actual content.

The box model allows us to place a border around elements and space elements in relation to other elements.

IDs vs. Classes

ID -- Should only apply to one element on a webpage (like #top -- there is only one top of the page).

Can be used to link to different parts of a single page. (Example: npr.org).


                  <p id="this_ID">
              

                  #this_ID { color: red; }

Class -- Lots of elements can have the same class.

Ex. There can be many warnings on one webpage.


                  <p class="this_class">
                  <div class="this_class">
                  <a href="..." class="this_class">
              

                  .this_class { color: #333; }

IDs vs. Classes

Another way to think about it:

Like a department store, you just have one women's department, one men's department, one kids' department. These departments are like ID's.

Inside each department are all kinds of products: sweaters, shirts, socks, in all colors and sizes. There are multiples of these, so these are like classes.

  • Any other elements, such as paragraphs or lists, will wrap around the floated element.

Z-index

When you use positioning to move elements out of the normal flow of content, elements can overlap. You can change the order of overlapping with z-index.

The element with highest z-index goes on top.

Pseudo-classes & CSS for links

Changing the format of a link when you hover over it is accomplished by using pseudo-classes.

CSS pseudo-classes are used to add special effects to some selectors.

Syntax:


              selector:pseudo-class {
                property:value;
              }
Example: 

                a:hover {
                  text-decoration: none;
                }

Pseudo-classes & CSS for links



              a:link    {color: #FF0000;} /* unvisited link */
              a:visited {color: #00FF00;} /* visited link */
              a:focus   {color: #FF00FF;} /* link currently in focus */
              a:hover   {color: #FF00FF;} /* mouse over link */
              a:active  {color: #0000FF;} /* selected link */

Note: a:hover MUST come after a:link and a:visited
in the CSS definition in order to be effective!

Note: a:active MUST come after a:hover in the
CSS definition in order to be effective!

Let's Develop it!

              a:focus   { color:#000; }
